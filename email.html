Headless agents can sign in to canopy-web.

A pre-shared token now mints a Django session via POST /api/auth/e2e-login/. From there gstack drives every OAuth-gated page the way you do — the screenshot below was taken that way.

Token in, session cookie out

Send {email, token}. If the token matches CANOPY_E2E_AUTH_TOKEN, a service user is created or reused and logged in; the response sets sessionid. The endpoint registers only when the env var is non-empty (404s otherwise) and emails must match the allowed domain. Sessions carry a _canopy_e2e_session marker so they can be filtered or revoked separately from human OAuth sessions.
