## Vision

### 10x Check
The system doesn't wait for you to bring conversations to it — it finds conversations worth extracting. It monitors Slack channels, open claw sessions, and API traffic, surfacing: "This thread between Neal and Eva could become a CRISPR analysis skill." The workspace becomes the center of a self-driving skill extraction flywheel. (Deferred to V2 — validate workspace first.)

### Platonic Ideal
Beth opens the web app Monday morning. Dashboard shows: "3 conversations from last week could become reusable skills" with confidence scores. "The CRISPR skill has been run 12 times. Eval score improved from 6.2 → 8.1." She clicks a suggestion, spends 15 minutes co-authoring, publishes. Neal gets notified: "New skill available: Partnership Opportunity Assessment." The feeling: "The system is always working for me. It turns our team's best thinking into everyone's capability."

## Key Learning: Reef Failure
Previous attempt to build skill+eval infrastructure via reef for open claws failed. Hypothesis: too hard to both build skills AND understand if/how they work with proven evals. This approach cracks it by:
1. Making eval extraction a first-class part of skill co-authoring (not an afterthought)
2. Making eval results visible via leaderboard (not hidden in logs)
3. Making the eval improvement story the primary internal selling point
